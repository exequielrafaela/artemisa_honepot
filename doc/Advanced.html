Advanced configuration¶

fingerprint.conf¶

This file inside /fingerprint/ is used to list the tuples application=fingerprint. The parameter fingerprint is searched in the SIP message, so that Artemisa determines that the analyzed message was created by the tool application.

For example, if you desire to tell Artemisa that the pattern “User-Agent: Twinkle/1.4.2” indicates that message was originated from Twinkle softphone, you may set the configuration file as:

Twinkle=Twinkle/1.4.2

Another example could be:

SIPVicious=friendly-scanner

Flooding armor¶

The threshold for the flooding armor is the parameter max_calls in artemisa.conf file. When flooding is detected, the file /scripts/on_flood.sh is executed. The administrator can modify the script in order to define firewall rules to deny the flooder IP.
